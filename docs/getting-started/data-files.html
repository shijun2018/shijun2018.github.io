数据文件

NexT 使用文档 – 开始使用 – 数据文件

当前, 通过 pulling 或下载新的发布版本更新 NexT 主题并不顺利. 通过git pull升级NexT主题时，它经常会陷入冲突状态, 或者在升级到新版本时，需要手动合并配置.

目前，NexT鼓励用户将一些选项存储在 站点配置文件 (hexo/_config.yml)中，将其他选项存储在 主题配置文件 (next/_config.yml)中. 此方法适用，但有一些缺点:

  1. 配置分为两部分
  2. 用户可能会感到困惑，应该选择哪个位置

为了解决此问题，NexT将利用 Hexo 数据文件. 由于Hexo 3中引入了数据文件，因此您需要将Hexo升级到3.0（或更高版本）才能使用此功能。

如果您更喜欢Hexo 2.x，仍然可以使用旧方法进行配置。NexT仍与Hexo 2.x兼容（但可能会出现错误）。

这样，您的所有配置都位于 站点配置文件中.
您无需编辑 主题配置文件 或创建任何新文件.

如果新版本中有任何新选项，则只需从 主题配置文件, 复制这些选项，然后粘贴到 站点配置文件中，然后将其值设置为所需的值即可.

用法

  1. 检查是否不存在 hexo/source/_data/next.yml 文件 (如果存在则将其删除).
  2. 将所需的NexT主题选项从 主题配置文件 复制到 站点配置文件, 然后
    2.1. 将所有这些设置向右移动两个空格 (在Visual Studio Code中：选择所有字符串, CTRL + ]).
    2.2. 在所有这些设置之上添加参数 theme_config: .

通过这种方式，您可以将所有配置放在一个位置 (hexo/source/_data/next.yml).
您无需编辑 主题配置文件 (next/_config.yml).
但是option可能无法准确地处理所有hexo外部库及其附加选项 (例如, 只能在默认hexo配置中读取模块选项hexo-server).

如果新版本中有任何新选项，则只需从 主题配置文件中复制这些选项，然后粘贴到 hexo/source/_data/next.yml 并设置其值即可.

用法

  1. 请确保您使用的是Hexo 3（或更高版本）.

  2. 在hexo/source/_data 目录创建一个next.yml文件 (_data如果不存在，则创建目录).

    在该步骤之后，有 2个变体, 只需 选择其中一个 然后 继续下一步即可.

    • override: false
    • override: true

    1. 在 next.yml文件中 override 必须未定义或设置为 false.
    2. 将 所需的选项 从 站点配置文件 和 主题配置文件 复制到 hexo/source/_data/next.yml中.

    1. 在 next.yml 文件中 override 设置为 true.
    2. 将 所有NexT主题选项 从 主题配置文件 复制到 hexo/source/_data/next.yml.

  3. 然后, 在主 站点的配置文件中需要定义 theme: next 选项 (如果需要还有, source_dir: source).
  4. 使用标准参数启动服务器，生成或部署 (hexo clean && hexo g -d && hexo s).

NexT 使用 lodash/merge 合并来自 next.yml的配置, 这可能导致菜单项顺序错误。由于lodash/merge正在使用深度复制，因此即使您设置菜单项的顺序，它们任然遵循_config.yml中的默认顺序.
