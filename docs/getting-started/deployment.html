部署

NexT 使用文档 – 开始使用 — 部署

本地部署

  1. 本地修改文件.
  2. 本地化测试: hexo clean && hexo g && hexo s.
  3. 部署: hexo g -d.

如果你使用 macOS 配合 hexo-deployer-git, 请确保 lib/ 目录没有被git忽略 (例如， check ~/.gitignore_global). 否则，部署后可能会丢失一些必要的文件.

持续集成

  • 直接在线编辑文件，立即生效
  • 自动部署，同时部署到多个位置
  • Travis CI
  • Gitlab CI

Travis CI使您的团队可以放心地测试和发布您的应用程序. 它为每个人以及各种规模的项目和团队而构建，支持开箱即用的20多种不同语言，包括Javascript和Node.js，Ruby，PHP，Python，Mac / iOS以及Docker，同时使您可以完全控制在构建环境中对其进行自定义以满足您的需求。

  • 获得权限 →
  • 配置 .travis.yml

有两种方法获取Travis CI的必要权限。部署密钥具有高安全性的优势，而访问令牌具有更加灵活的优势。

  • 访问令牌
  • 部署密钥

此方法适用于具有私有子模块的仓库

  • 在托管博客源的资源库中创建的空分支 source.
  • 获得令牌: Settings -> Developer settings -> Personal access token -> Generate new token. 根据实际情况设置访问权限。应该注意的是，访问令牌仅在该页面上显示一次，并且应该被复制，否则只能重新生成。
  • 使用 github 登录进入 Travis CI website. 您可以在当前的github帐户中找到仓库，选择要部署的仓库，然后单击“设置”。填写github生成的个人访问令牌，然后选择Travis需要监视仓库的分支。

部署密钥是在您的存储库中设置的SSH密钥，用于授予客户端对存储库的只读访问权限（如果需要，还可以授予r / w）。此方法适用于大多数公共博客存储库，并且存储库中没有私有子模块。

  • 在托管博客源的资源库中创建的空分支 source.

  • 生成一个ssh密钥

  • 在您的存储库中添加部署公钥，并将其删除

  • 使用Travis命令加密私钥，并将其添加到git

GitLab 提供一个持续集成服务和页面服务. 如果你添加一个 .gitlab-ci.yml 到你仓库的根目录, 并将GitLab项目配置为使用Runner, 则每次提交或推送都会触发CI管道. 文件 .gitlab-ci.yml 告诉 GitLab runner 做什么. 默认情况下，它运行带有三个阶段的管道：构建，测试和部署. 您不需要使用所有三个阶段；没有工作的阶段将被忽略。最后，您的网站将自动在GitLab Host上发布。

  1. 添加 .gitlab-ci.yml 到您的存储库的根目录，并对其进行配置.
  2. Upload scaffolds, source, themes, .gitignor, .gitlab-ci.yml, _config.yml, and package.json to your Gitlab repository.

现在，您可以在https://yourname.gitlab.io/project类似于GitHub的位置访问静态网站. 更多的GitLab页面配置在这里.

当然，您也可以在GitHub Pages或其他页面服务上使用静态网站。有两种配置方法 .gitlab-ci.yml:

  • HTTPS
  • SSH
  • 获取访问令牌: Settings → Developer settings → Personal access token → Generate new token. 根据实际情况设置访问权限。应该注意的是，访问令牌仅在该页面上显示一次，并且应该被复制，否则只能重新生成.
  • 点击 SETTINGS-CI/CD → Variables in Gitlab, 将访问令牌定义为自定义变量 GITHUB_ACCESS_TOKEN. 或者设置编码存储库的 USERNAME PASSWORD 变量.
  • 配置 .gitlab-ci.yml: 仅在此文件的末尾添加部署阶段

部署密钥是在您的存储库中设置的SSH密钥，用于授予客户端对存储库的只读访问权限（如果需要，还可以授予r / w）。此方法适用于大多数公共博客存储库，并且存储库中没有私有子模块。

  • 生成部署密钥

  • 在Gitlab中单击 SETTINGS-CI/CD → Variables , 复制私钥的内容并将其定义为自定义变量 DEPLOY_PRIVATE_KEY.

  • 配置 .gitlab-ci.yml: 仅更新脚本 before_script

变量不会被屏蔽，如果明确要求，变量的值可以显示在作业日志中。因此，请确保gitlab管道只能由您自己查看。
